# Using ping for Network Troubleshooting

## Purpose

The `ping` command is used to test connectivity between devices and verify internet/network access.

---

## Commands Practiced

### Test Internet Connectivity

```bash id="4z7v1j"
ping 8.8.8.8
```

Used to verify whether internet connectivity is available through direct IP communication.

---

### Test DNS Resolution

```bash id="c2l5g8"
ping google.com
```

Used to confirm whether DNS resolution is functioning correctly.

---

### Test Local Network Connectivity

```bash id="k8x2p4"
ping 192.168.1.1
```

Used to verify communication with the local router or gateway.

---

## Troubleshooting Scenarios

### Scenario 1 — Internet Not Working

* Successful ping to router
* Failed ping to external IP
* Diagnosed internet connectivity issue

### Scenario 2 — DNS Failure

* Successful ping to 8.8.8.8
* Failed ping to domain name
* Diagnosed DNS resolution issue
